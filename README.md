## 🔄 How I Work in Scrum

### Sprint Ceremonies

| Ceremony | Duration | My role | Output |
|----------|----------|---------|--------|
| **Sprint Planning** | 2–4h | Facilitator + Backlog coach | Sprint goal, committed backlog, capacity plan |
| **Daily Stand-up** | 15 min | Facilitator + Impediment tracker | Impediment log updated, blockers escalated |
| **Sprint Review** | 1–2h | Facilitator + Stakeholder bridge | Demo feedback, acceptance validated |
| **Retrospective** | 1–1.5h | Facilitator | Start/Stop/Continue, action items with owners |
| **Backlog Refinement** | Ongoing | Co-author with PO | Refined user stories, acceptance criteria, estimates |

---

### Backlog Prioritization — MoSCoW + Story Points

I use **MoSCoW** for priority classification and **Fibonacci Story Points** estimated with Planning Poker.

**Sample Sprint Backlog (AI Tourism Platform — Sprint 4):**

| ID | User Story | MoSCoW | SP | Status |
|----|-----------|--------|----|--------|
| US-14 | As a hotel manager, I want to query the LLM via natural language to get occupancy forecasts | **Must** | 8 | ✅ Done |
| US-15 | As a TUI analyst, I want to export forecast data to CSV with one click | **Must** | 3 | ✅ Done |
| US-17 | As a Barceló admin, I want a dashboard showing real-time team velocity | **Should** | 5 | 🔄 In Progress |
| US-19 | As a user, I want to switch the UI language between EN/ES/PT | **Could** | 3 | ⬜ To Do |
| US-22 | As an admin, I want AI-generated competitor pricing reports weekly | **Won't** | 13 | ⛔ Deferred |

**Sprint capacity:** 24 SP committed (Must + Should items only). Won't items moved to next sprint backlog.

---

### Impediment Removal Process

```
Blocker identified (daily stand-up or async)
        ↓
Logged in Jira impediment board (same day)
        ↓
Can SM resolve? ──Yes──→ Resolved within 24h
        ↓ No
Escalated to PO or stakeholder (SLA: 24h response)
        ↓
Tracked until resolution → Sprint closure report
        ↓
Patterns reviewed in Retrospective → Process improvement
```
