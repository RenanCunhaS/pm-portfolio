How I Work in Scrum

Sprint Ceremonies

01

Sprint Planning

Backlog refinement with PO. MoSCoW + Story Points. Sprint goal defined. Capacity checked.

02

Daily Stand-up

15 min max. Yesterday / Today / Blockers. Impediment log updated in real time.

03

Sprint Review

Demo to stakeholders. Acceptance criteria validated. Velocity and defect rate reported.

04

Retrospective

Start / Stop / Continue. Action items assigned with owners and due dates.

05

Backlog Refinement

User stories broken down. Acceptance criteria written. Estimates calibrated.

Sample Sprint Backlog — AI Tourism Platform (Sprint 4)

Prioritized using MoSCoW. Story Points estimated with Planning Poker (Fibonacci).

# User Story MoSCoW SP Status Acceptace Criteria (key)
US-14 As a hotel manager, I want to query the LLM via natural language to get occupancy forecasts Must 8 ✓ Done Response ≤ 3s · Accuracy ≥ 85% · Fallback on timeout
US-15 As a TUI analyst, I want to export forecast data to CSV with one click Must 3 ✓ Done CSV includes headers · UTF-8 encoded · Works on mobile
US-17 As a Barceló admin, I want a dashboard showing real-time team velocity per sprint Should 5 ⟳ In Progress Auto-refresh every 30s · Burndown visible · Filters by team
US-19 As a user, I want to switch the UI language between EN/ES/PT Could 3 ⊙ To Do All strings translated · Persisted in session · RTL not required
US-22 As an admin, I want AI-generated competitor pricing reports weekly Won't 13 ⊗ Deferred Moved to Sprint 5 backlog pending data contract sign-off
